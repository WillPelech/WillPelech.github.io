Tax Agent built without AI

2026 · Python · github.com/WillPelech/tax_agent

An AI agent that does your taxes — built entirely by hand, no AI assistance in writing the code. It uses a ReAct (Reason + Act) loop to reason through tax logic, call tools via MCP, and produce results in an isolated sandbox environment.

[ screenshots / demo images coming soon ]

How it works

The agent follows the ReAct pattern: it alternates between reasoning about the current state and taking actions by calling tools exposed through an MCP server. Each action's result is fed back into the next reasoning step until the agent reaches a final answer.

Tool execution happens inside a sandboxed VM so nothing runs on your host machine directly. The sandbox is bootstrapped via shell scripts and communicated with over a local URL.

Architecture

agent/
  agent.py      # LLM client factory + ReAct loop
  client.py     # LLMClient base class + provider implementations
  prompt.py     # system and user prompts
  registry.py   # MCP tool registry
mcp_server/     # MCP server exposing tax tools to the agent
scripts/        # VM / sandbox bootstrap
main.py         # entrypoint

Provider-agnostic design

The client layer is fully swappable. Switch providers by editing model-config.yaml: 

model: "qwen3:0.6b"
provider: "Qwen"
sandbox_url: "http://localhost:8080"

Adding a new provider means subclassing LLMClient in agent/client.py, implementing chat(self, message: str) -> str, and adding a branch in buildClient.

Running locally — no API key needed

The whole thing runs on a small local model via Ollama, so you don't need an OpenAI key or any external service. 

ollama pull qwen3:0.6b
git clone https://github.com/WillPelech/tax_agent
cd tax_agent
uv sync
uv run main.py

Why no AI?

I wanted a clear understanding of every layer — the ReAct loop, the MCP tool protocol, the sandbox boundary, and the provider abstraction. Writing it from scratch forced me to actually understand each piece rather than accepting generated boilerplate. Coming off my internship at Datadog where I worked on AI agent infrastructure at scale, it was a good exercise to build the full stack myself.
