AI Valet Robot

2026 · Python · Raspberry Pi · Computer Vision · github.com/WillPelech/senior_design_final

For my senior design project I built an autonomous valet robot — a Raspberry Pi-powered rover that navigates a track using color detection, picks up toy cars from designated parking spots with a servo lift, and delivers them to an exit point. The whole thing is controllable from a web UI over your local network.

What it does

The robot sits at HOME until you trigger a mission. You press 1 or 2 (or click the web UI) and it navigates to the corresponding parking spot, lifts the car onto a platform using two FS90 micro servos, drives to the EXIT, lowers the car, then returns home. All autonomously.

Navigation is purely vision-based. There are no encoders, no GPS, no IMU. A forward-facing USB webcam detects colored squares on the walls — green for HOME, purple for EXIT, blue for Parking Spot 1, red for Parking Spot 2. The robot spins in place until it sees the target color, then uses PID steering on the horizontal offset from frame center to drive toward it.

State machine

IDLE → NAVIGATE → LIFTING → DELIVER → LOWER → RETURN → IDLE

Each state has one job. NAVIGATE line-follows to the target spot. LIFTING raises the servo platform under the car. DELIVER drives to EXIT. LOWER drops the car. RETURN goes home. If you hit SPACE at any point it hard-stops. The state machine made the logic easy to reason about and easy to debug — when something went wrong I knew exactly which state to look at.

PID steering

The steering controller runs on the x-error — the horizontal pixel offset of the detected target from the center of the frame. Proportional term corrects the heading, derivative term damps oscillation when the robot overshoots. All tunable constants live in config.py so I could adjust them without touching the control logic.

Web UI

The web UI runs a Flask server on your laptop. It SSH-es into the Pi and uses tmux send-keys to inject keypresses into the robot session running in a detached tmux session. Clunky but effective — you just open localhost:5000 and click buttons to send missions from any device on the network.

What was hard

Color detection under variable lighting was the main challenge. The colored squares needed to be distinct enough that the HSV thresholds didn't accidentally trigger on each other, but robust enough to work under different room lighting. I ended up tuning the thresholds empirically and adding a minimum area filter so small color noise didn't count as a detection.

The servo lift timing also took iteration. The platform needed to raise slowly enough not to knock the car off, but fast enough that the robot didn't sit still long enough to drift off the line. The final pulse widths are in config.py: SERVO_PULSE_DOWN_US = 1000, SERVO_PULSE_UP_US = 1800.
